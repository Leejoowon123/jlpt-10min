# 실제 브라우저 수동 QA 체크리스트

`smoke.mjs` 와 `qa.mjs` 는 데이터 무결성·로직·DOM 시나리오를 자동 검증하지만
**jsdom 은 레이아웃·실제 TTS/STT 를 실행하지 않는다**. 다음 항목은 실제 브라우저
(Chrome / Edge / Safari) + 모바일 폭 (360 × 740 권장) 에서 한 번씩 직접 확인한다.

## 환경
- 브라우저: Chrome 또는 Edge 최신, 모바일 에뮬레이션 360×740 (DevTools `Toggle device toolbar`).
- 정적 서버: `python -m http.server 5173` (`D:\TEST\` 에서 실행) → `http://localhost:5173`.
- 학습 데이터 초기화가 필요하면 DevTools → Application → Local Storage → `jlpt10min:v1` 삭제.

## 0-T. 테마 / 스토리 본문 3단 구조 / compact player (라운드 18)

- [ ] 각 문단이 일본어(후리가나 포함) → 로마자(이탤릭) → 한국어 해석 순서로 표시.

- [ ] "전체 해석" 보조 탭도 동작.

### compact player
- [ ] 하단 플레이어가 [⏮][▶][⏭] 위치 상태 [0.75x][1x][1.25x] 한 줄(최대 2줄)로 표시.
- [ ] 360px 폭에서 줄바꿈은 자연스럽고 가로 스크롤 없음.
- [ ] 본문 마지막 문단이 compact 플레이어에 가려지지 않음 (padding 120px).
- [ ] 재생 중 상태 라벨이 "재생", 일시정지 버튼 ⏸ 로 전환.

### cover 이미지
- [ ] 雨の朝 / 春の日曜日 / 駅で道を聞きました 목록 row 에 SVG 썸네일 표시.
- [ ] 상세 상단에 cover 표시. cover 없는 이야기는 기존 그대로 (fallback).
- [ ] 잘못된 src 시 이미지가 조용히 사라지고 화면이 깨지지 않음.
- [ ] 새 이미지 추가 시 docs/asset-licenses.md 에 기록되어 있는지 확인.

- [ ] 본문(이야기 탭)에 한자 위 ruby 후리가나 (설정 ON 시) 표시.
- [ ] **탭 4개** 노출: 이야기 / 핵심 단어 / 문법 / 해석. 기본 탭은 "이야기".

- [ ] **개별 문단 듣기** — 문단마다 `🔊 이 문단 듣기` 버튼 클릭 시 그 문단만 재생되고 다음 문단으로 자동 진행하지 않는다. aria-label 이 "이 문단 듣기" 인지 DevTools 확인.

- [ ] **문단 사이 약 0.7초 쉬는 시간** — 한 문단이 끝나고 다음 문단이 시작되기 전에 명확한 일시 정지가 들린다.

- [ ] **이전/다음 버튼** ⏮ ⏭ — 클릭 시 active 문단이 한 칸씩 이동. 재생 중이면 새 문단부터 다시 재생.

- [ ] **위치 표시** `현재 / 총` 이 다음 문단으로 이동할 때마다 갱신.
- [ ] **상태 표시** "정지 / 재생 중 / 문단 재생 중" 이 모드에 맞게 갱신.
- [ ] **핵심 단어 탭** — 단어 카드 목록 (최소 3개) 표시.
- [ ] **문법 탭** — 관련 문법 패턴 목록 (최소 1개) 표시.
- [ ] **해석 탭** — 전체 한국어 번역 노출. 다시 이야기 탭으로 돌아가면 본문에 한국어 없음.
- [ ] **본문 안 인라인 하이라이트** — 핵심 단어가 본문 안에서 직접 점선 박스로 표시된다 (.story-inline-hl). 터치/클릭 시 해당 문단 아래 패널에 "단어 (읽기) — 의미" + 🔊 발음 듣기 + (vocabId 있을 때) "단어 학습 →" 버튼.
- [ ] 같은 인라인 하이라이트 두 번 누르면 패널이 닫힘.
- [ ] 인라인 하이라이트 안의 후리가나 ruby/rt 가 점선 박스와 겹치거나 잘리지 않는다.
- [ ] **단어 학습 →** 클릭 시 학습 > 단어 > 찾아보기 화면으로 이동하면서 해당 단어가 검색 prefill 되어 row 가 바로 보인다.
- [ ] **핵심 단어 탭** 의 각 row 우측에 "단어 학습 →" 버튼이 있다. 클릭 시 동일하게 학습 화면으로 이동.

### 0-3-2. 실제 Chrome / Edge — TTS 시퀀스 (수동)

- [ ] 재생 중 **속도 0.75x → 1.25x** 로 바꾸면 **현재 문단은 그대로 끝까지 재생** 되고 다음 문단부터 속도가 바뀌어 들린다 (Web Speech API 한계 안내).
- [ ] 재생 중 **다음 ⏭** 클릭 → 즉시 다음 문단 처음부터 재생, active 표시도 즉시 이동.
- [ ] 재생 중 **상단 뒤로(← 단편 소설 목록)** / 다른 탭 진입 → 음성 즉시 멈춤. 백그라운드에서 다음 문단이 추가로 들리지 않는다.
- [ ] 마지막 문단 자동 정지 후 상태 표시가 "정지" 로 바뀐다.
- [ ] **scrollIntoView** — 5문단 정도의 단편 소설에서 next 클릭으로 진행하면 active 문단이 화면 중앙 근처로 자동 스크롤된다.
- [ ] **하단 플레이어 겹침** — 모바일 360px 에서 본문의 마지막 문단 텍스트가 하단 고정 플레이어에 의해 가려지지 않는다 (`.has-story-player { padding-bottom }` 효과).
- [ ] **route 이동 cleanup** — 본문 재생 중 뒤로(`← 단편 소설 목록`) / 다른 이야기 진입 / 탭 이동 시 음성이 즉시 멈춘다. 백그라운드 타이머 잔존 없음.
- [ ] **모바일 폭 360 px** — 본문 line-height 가 답답하지 않고, ruby 와 본문이 겹치지 않으며, **하단 고정 플레이어가 본문의 마지막 줄을 가리지 않는다** (`.has-story-player` padding-bottom 적용). 플레이어 버튼이 한 줄을 넘어가도 자연 줄바꿈.

## 1. 홈 화면
- [ ] 페이지 진입 시 상단에 **오늘의 10분** 카드 표시. 시작 버튼 동작.
- [ ] 카드 우상단에 **후리가나** 체크박스 토글 노출 + 기본 체크 ON.
- [ ] 3-grid 통계 (오늘 복습 / 실패 노트 / 자주 볼 단어) 표시.
- [ ] **회화 준비도** 카드 표시. N5 6개 주제 모두 노출, 미학습 상태에선 `학습 필요` 라벨.
- [ ] **학습 영역** 4버튼 (단어/문법/독해/청해) 동작.
- [ ] **복습 / 비교** 2버튼 동작.

## 1-1. 학습 설정 카드 (수동 확인)
홈 화면 "오늘의 10분" 카드 아래 별도 카드 (`#settingsCard`). 다음 3개 컨트롤이 보인다.

- [ ] **후리가나 표시** 체크박스 + 짧은 설명 ("한자 위 읽기 보조")
- [ ] **이미지 카드 단계형 학습** 체크박스 + 짧은 설명 ("노출 → 회상 → 확인 → 퀴즈")
- [ ] **회상 시간** 라벨 + [3초][5초][7초] 세그먼트 버튼. 현재 선택값에 active 강조.
- [ ] 학습량 현황 카드는 보이지 않는다 (정책 유지).
- [ ] 모바일 360px 폭에서 카드 안의 컨트롤들이 줄바꿈되어도 자연스럽고 가로 스크롤 없음. 세그먼트 버튼 3개가 한 줄에 들어가지 않을 경우 다음 줄로 자연 줄바꿈.

### 후리가나 토글 동작
- [ ] 홈 카드의 **후리가나** 체크박스 ON 상태에서 단어 예문/문법/독해/청해/회화/문법비교/복습 화면 진입 시 한자 위에 작은 히라가나가 보인다.
- [ ] 체크박스 **해제(OFF)** → 토스트 "후리가나 표시 OFF" 표시. 같은 화면 재진입 시 히라가나 표기 없이 한자 원문만 표시. **일본어 원문은 빠지지 않고 그대로 보인다.**
- [ ] 체크박스 다시 **체크(ON)** → 토스트 "후리가나 표시 ON". 화면 재진입 시 ruby/rt 다시 표시.
- [ ] OFF 상태에서 페이지 **새로고침** 후에도 OFF 가 유지된다 (localStorage `jlpt10min:v1` 의 `settings.furiganaEnabled` 값 확인).
- [ ] OFF 상태에서 **단어 4지선다** 풀이 — 선택 전 한국어 의미/번역/연상 텍스트는 여전히 노출되지 않는다 (정답 누출 방지 정책 유지).
- [ ] OFF 상태에서 **이미지 카드 thinking** — 단어/읽기/연상 모두 미노출 (변동 없음).
- [ ] **한자 카드 reveal 후 히라가나** (예: 日 → ひ) 는 후리가나 토글이 OFF 여도 그대로 표시된다. (한자 카드의 hiragana 필드는 별도 학습 정보)
- [ ] 모바일 폭 **360px** 에서 ruby line-height 가 충분해 윗줄/아랫줄 글자 겹침 없음. 예문 단락 사이 간격이 너무 좁지 않은지 확인.

### 이미지 카드 단계형 학습 토글 동작
- [ ] 토글 OFF → 토스트 "단계형 학습 OFF". 학습 > 단어 > 이미지 카드 진입 시 곧장 5/5 퀴즈로 시작 (1/5 보기 단계 없음).
- [ ] 토글 ON 복귀 → 토스트 "단계형 학습 ON". 카드 진입 시 1/5 보기 단계부터 시작.
- [ ] 새로고침 후에도 마지막 설정값 유지.

### 회상 시간 동작
- [ ] [3초] 선택 → 1/5~3/5 진행 시 recall 단계 카운트다운이 "3초 후 자동 확인"으로 시작.
- [ ] [5초] 선택 + 단어 카드 진입 → recall 단계에서 카운트다운이 **"5초 후 자동 확인"** 으로 시작. 5초가 지난 뒤 자동으로 4/5 확인 단계로 전환.
- [ ] [7초] 선택 → recall 카운트다운이 **"7초 후 자동 확인"** 으로 시작. 7초 후 자동 전환.
- [ ] "바로 확인" 버튼은 어떤 회상 시간이어도 즉시 4/5 로 점프.
- [ ] 모바일 폭 360px 에서 [3초][5초][7초] 세그먼트 버튼이 가로 영역을 벗어나면 자연 줄바꿈, 가로 스크롤 없음.

## 4. 단어 이미지 카드 — 단계형 학습 플로우 + 발음 듣기
기본값 `settings.vocabImageWarmupEnabled = true` 에서 다음 5단계가 순차 진행된다.

### 4-1. 노출 1 (`1/5 보기`)
- [ ] 진입 시 헤더 우측에 `1/5 보기` 라벨이 보인다.
- [ ] 이미지(아이콘) 아래에 단어 + reading + 한국어 뜻이 한 번에 노출.
- [ ] 🔊 **발음 듣기** 버튼이 보이고, 진입 직후 자동 재생을 시도한다. 자동 재생이 막혀도 학습 흐름이 멈추지 않는다.
- [ ] 일본어 voice 가 없으면 **"일본어 TTS 음성을 찾을 수 없습니다."** hint 가 보인다.
- [ ] 4지선다(`.choice`)는 **아직 보이지 않는다.**
- [ ] "다음 →" 클릭 시 노출 2 단계로 이동한다.

### 4-2. 노출 2 (`2/5 한 번 더`)
- [ ] 헤더 라벨이 `2/5 한 번 더` 로 바뀐다.
- [ ] 단어 + reading + 뜻이 다시 노출된다.
- [ ] 🔊 발음 듣기 버튼이 보인다.
- [ ] "다음 →" 클릭 시 회상 단계로 이동.

### 4-3. 회상 (`3/5 떠올리기`)
- [ ] 이미지만 보이고 단어/reading/뜻/예문/연상 텍스트가 **모두 사라진다** (브라우저 DevTools 의 Elements 패널에서도 부재 확인).
- [ ] "이 이미지에 어울리는 단어를 떠올려 보세요." 안내가 보인다.
- [ ] **3초 카운트다운** + 진행 바가 동작한다 (이미지 라벨 빈칸, 발음 듣기 버튼은 없음).
- [ ] 3초가 지나면 자동으로 확인 단계로 전환.
- [ ] **"바로 확인 →"** 버튼 클릭 시 즉시 확인 단계로 건너뛰기 가능.
- [ ] 회상 중 "다음"이나 탭바로 화면을 벗어났다 돌아오면 카운트다운이 잔존하지 않는다.

### 4-4. 확인 (`4/5 확인`)
- [ ] 헤더 라벨 `4/5 확인`.
- [ ] 단어 + reading + 뜻 + 예문 + 한국어 해석 + 연상 텍스트가 모두 노출.
- [ ] 🔊 발음 듣기 + 🔊 예문 듣기 버튼이 모두 동작.
- [ ] **"퀴즈로 →"** 클릭 시 퀴즈 단계.

### 4-5. 퀴즈 (`5/5 퀴즈`) — 기존 동작
- [ ] 4지선다 노출.
- [ ] 단어/reading/뜻/예문/연상 텍스트가 다시 **숨겨진다** (정답 단어는 4지선다 안에만 존재).
- [ ] 발음 듣기 버튼의 `aria-label` / `title` 에 정답 단어가 **포함되지 않음** (Accessibility tree 확인).
- [ ] 정답 선택 시 .explain 표시 + 예문/연상/번역 모두 노출 + 다음 버튼 활성화.
- [ ] 오답 선택 시 토스트 "실패 노트에 추가되었습니다" 표시.

### 4-X. 일반 확인 사항
- [ ] 다음 버튼 클릭 시 진행 중 음성이 자동 정지.
- [ ] **퀴즈 답변 전까지 통계가 증가하지 않는다.** (홈 → "오늘의 10분" 카운터를 확인하면 expose/recall/confirm 전이는 카운트되지 않음.)
- [ ] **모바일 폭 360 px** — 이미지 / 단어 / 뜻 / 버튼 / 단계 라벨이 한 줄에서 잘림 없이 표시. 카운트다운 진행 바가 카드 폭을 벗어나지 않음.
- [ ] 노출 1/2 와 확인 단계의 단어/뜻이 너무 작게 표시되지 않음 (font-size 13~16 px).
- [ ] LocalStorage 의 `jlpt10min:v1 → settings.vocabImageWarmupEnabled = false` 로 직접 바꾼 뒤 새로고침 → 이미지 카드 진입 시 곧장 5/5 퀴즈로 진입 (호환 모드 확인).
